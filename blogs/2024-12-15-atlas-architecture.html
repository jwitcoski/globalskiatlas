AWS Architecture for Ski Resort Atlas

December 15, 2024

Serverless Architecture Overview

The Global Ski Atlas leverages a sophisticated serverless AWS architecture to collect, process, and serve data about ski resorts worldwide. This architecture minimizes operational overhead while providing scalability and reliability.

Data Ingestion Pipeline

The left side of the diagram shows our data ingestion pipeline, which starts with OpenStreetMap data and processes it through a series of AWS Lambda functions orchestrated by Step Functions:

  • step1_query_osm: Fetches winter sports areas from OpenStreetMap
  • process_resort_queue: Distributes processing work through SQS
  • check_queue_status: Monitors processing progress
  • step4_download_resort_data: Consolidates processed resort data

Data Storage

The processed resort data is stored in two primary locations:

  • DynamoDB: Stores resort metadata and statistics for fast retrieval
  • S3 Bucket: Stores larger resort assets like map files and images

API Layer

The center of the diagram shows our API layer, which consists of two primary API Gateway endpoints:

  • Resort Selection API: Provides endpoints for fetching countries, states, and resorts
  • AI Description API: Generates natural language descriptions of resorts using Amazon AI services

User Interface

The right side shows how the browser interacts with our architecture:

  • resortSelector.js: Handles the dropdown selection interface
  • aiWriter.js: Manages AI description generation and display
  • Map Visualization: Renders resort boundaries, lifts, and trails

Future Enhancements

In upcoming releases, we plan to enhance the architecture with:

  • Real-time weather data integration
  • User accounts and saved favorites
  • Advanced filtering options for resort selection
  • Mobile app with offline map capabilities

This serverless architecture has allowed us to create a powerful, scalable application without managing servers, while keeping operational costs proportional to actual usage.
